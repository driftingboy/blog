网络编程模型：客户端-服务端

服务器端需要在一开始就监听在一个众所周知的端口上，等待客户端发送请求，一旦有客户端连接建立，服务器端就会消耗一定的计算机资源为它服务，服务器端是需要同时为成千上万的客户端服务的。

ip 和 port

ip 是网络世界的地址，可以定位到一台机器，但在一台计算机上是可以同时存在多个连接的，那么如何区分出不同的连接呢？这里就要引入端口概念

port 可以定位到计算机中的进程；

举个例子：

ip定位范围较广泛，就像你想找一个人通过ip只能找到他在的那栋楼，通过端口就像在那栋楼里通过工位号找到目标人物

一个连接可以通过客户端 - 服务器端的 IP 和端口唯一确定，这叫做套接字对;

子网掩码

ip由两部分组成

第一是网络（network），直观点说，它表示的是这组 IP 共同的部分，比如在 192.168.1.1~192.168.1.255 这个区间里，它们共同的部分是 192.168.1.0。

第二是主机（host），它表示的是这组 IP 不同的部分，上面的例子中 1~255 就是不同的那些部分，表示有 255 个可用的不同 IP。

比如上表中，第一行，是一个a类网络；可以看到它的host id size是24，也就是可以分配给 2的24次方个主机！而mask bits为8表示，2的8次方，也就是255.0.0.0

当然，子网掩码能接受任意个位，而不单纯是上面讨论的 8，16 或 24 个比特而已。比如：当我们只有两台主机，分配一个255.255.255.252（二进制位 11111111.11111111.11111111.11111100）也就是只有2的2次方个主机位。

子网掩码比如255.192.0.0看起来不直观，不知道网络位数；写成192.0.2.12/30这样更清晰，表示30位网络，2位主机；

域名系统

ip不利于记忆，域名（DNS）记录了网站和 IP 的对应关系；

tcp和udp

tcp/ip协议栈，或者说传输层，有两种协议：tcp、udp

tcp又被叫做字节流套接字（Stream Socket）,可靠的、双向连接的数据流；

udp又叫做数据报套接字（DataGram Socket）；1.广播或多播 2.联网游戏、视屏会议
