容器核心知识概述

發表於2021-06-30|更新於2021-08-22|云原生
|閱讀量:

什么是容器?

容器是一种轻量级、可移植、自包含的软件打包技术，由它打包的应用可以几乎在任何地方无差别的运行

自包含：指容器拥有程序所需要的基本运行时；

可移植：指容器具有跨平台的作用；

轻量级：指容器的大小非常轻量；

容器与虚拟机

这里我们把容器和虚拟机做一个比较

  • 轻量

    一个虚拟机通常的大小通常几十个GB,需要安装部署的应用、应用的依赖、完整的操作系统；

    一个容器通常大小几十MB，只需要安装部署的应用、应用的依赖，操作系统复用host os

  • 隔离性

    虚拟机的隔离性是优于容器的；容器共享host的kernel，所以内核的版本是无法修改的；

可以说容器牺牲了一些隔离性换取了轻量的特性。

为什么使用容器？

简单来说：它让应用程序具有了超强的可移植能力
容器具有 安全性、隔离性、可配额、便携性 的特性

现在的大部分应用程序，都是由多个服务构建、组织而成（MQ、DB、…）,这些服务都有自己的依赖，都可能部署到不同的硬件环境（共有云、私有云、虚拟机…）,这样排列组合下来会产生各种情况，需要开发人员开发时考虑运行环境的不同，运维人员也要根据不同的服务和运行环境做出不同配置。

容器就是抹平了由不同服务和平台带来的复杂度；

可以类比运输行业:

各种不同的物品可能需要不同的运输工具运输，而集装箱抹平了这一层的复杂度，无论是钢琴还是大米都统一放到各自的集装箱中。标准的集装箱可以适配各种交通工具，方便的在各个运输工具之间移动。

容器是如何运行的？

容器的架构

docker采用Client/Server架构。
核心组件如下

  1. docker 客户端：client

    最常用的client就是 docker命令，当然也可以通过 REST API 与服务器通讯.

  2. docker 服务器：daemon

    docker daemon负责创建、存储镜像，创建、运行、管理容器；不仅可以接收本地host的客户端请求，还可以接收远程客户端请求（需要修改配置文件，打开tcp监听）

  3. docker 镜像：images

    用于创建docker容器的只读模版；
    你可以把它看作一个类，而容器就是它生成的对象实例；

  4. docker 容器：containers

    docker 镜像运行的实例；按照软件的生命周期理解，镜像处于软件的构建、打包阶段，容器处于软件启动、运行阶段

  5. registry

    存放docker镜像的仓库，分为私有和公有；

    公有的比如 docker hub，私有库可以使用docker hub上提供的 registry 容器快速搭建；

（1）在本地寻找镜像，未找到

（2）在register（docker hub）拉取镜像

（3）远程镜像下载到本地了

（4）创建运行容器
