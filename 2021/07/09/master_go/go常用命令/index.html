go 命令

编译与运行

go build : go run : go install : go build 的关系

go generate

简介

参数说明：

  • run 正则表达式匹配命令行，仅执行匹配的命令
  • v 打印已被检索处理的文件。
  • n 打印出将被执行的命令，此时将不真实执行命令
  • x 打印已执行的命令

执行包下所有将被执行的命令

go generate runtime

打印包下所有将被执行的命令，不会真的执行

go generate -n ./...

输出如下：可以看到各种会执行的命令

使用方式

需在的代码中配置generate标记，格式如下：

go generate 可以像shell一样获取环境变量

并且在执行go generate时将会加入些信息到环境变量，可在命令程序中使用。

env desc
$GOARCH 架构 (arm, amd64, etc.)
$GOOS OS (linux, windows, etc.)
$GOFILE 当前处理中的文件名
$GOLINE 当前命令在文件中的行号
$GOPACKAGE 当前处理文件的包名
$DOLLAR

场景二 生成各种支持类型的队列

因为go中没有泛型，如果我们要实现一个什么都能存储的队列，除了使用interface，最安全的做法就是每种类型写一个队列实现，但是它们逻辑相同。

可以考虑使用 go generate + template 来实现，先为该逻辑编写模板，非常简单的队列逻辑
https://blog.schwarzeni.com/2020/01/07/go-generate-%E7%AE%80%E5%8D%95%E5%AE%9E%E7%94%A8/
