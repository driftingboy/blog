http code

code简述

code desc
1xx 表示处理的中间状态，还需要继续操作
2xx 表示报文已收到，而且成功处理
3xx 表示资源位置改动，需要客户端重新请求
4xx 表示客户端错误，比如参数错误
5xx 表示服务端错误，内部错误

客户端和服务端都需要正确的应用，处理这些错误码；客户端需要更具这些错误码知道请求是否被正确处理，知道下一步的应该做什么操作，所以服务端最好严格要求错误信息返回，如果需要处理，尽量返回一些有意义的code；

举例

  • 1xx
    比如客户端使用 Upgrade 头字段，要求在 HTTP 协议的基础上改成其他的协议继续通信，比如 WebSocket;如果服务端同意就会回一个101 Switching Protocols

  • 2xx
    200 ok

    204 No Content 和 200 一样都是成功不过不会携带body

    206 Partial Content是 HTTP 分块下载或断点续传的基础, 表示客户端只获取资源的部分数据时，如果请求成功，服务端返回206并且一般回content-range(Content-Range: bytes 0-99/2000)以便客户端确认。

  • 3xx

301 302 一个是永久重定向，一个是临时重定向；表示资源位置修改，回在响应头加入location明确表示要跳转的url；
如果是永久的改动，比如把服务从http切换为https，http的服务弃用；如果是临时的改动，比如服务器维护，把流量暂时引导到一个通知页面。

304 Not Modified 如果资源未修改，服务端会回一个304响应码，这时候客户端再重定向已到缓存的文件（即缓存重定向）。
其中涉及到http headerif-modify-since Etag 等等

  • 4xx

400 Bad Request是一个通用的错误码，表示请求报文有错误，但具体是为何错误没有说明；所以一般会在响应body里带上业务错误码和描述（如：参数错误 {code: 10001, desc: “param xxx miss”}）

403 Forbidden 403禁止实际上不是客户端的请求出错，而是表示服务器禁止访问资源。比如：请求资源敏感等等;

404 Not Found 表示服务器资源为找到;

405 Method Not Allowed 不允许使用某些方法操作资源，例如不允许 POST 只能 GET;

406 Not Acceptable 资源无法满足客户端请求的条件，例如请求中文但只有英文;

408 Request Timeout请求超时，服务器等待了过长的时间;

409 Conflict多个请求发生了冲突，可以理解为多线程并发时的竞态;

413 Request Entity Too Large请求报文里的 body 太大;

414 Request-URI Too Long请求行里的 URI 太大;

429 Too Many Requests客户端发送了太多的请求，通常是由于服务器的限连策略;

  • 5xx

500 internal error 内部错误，一般不会给出详细错误信息，是对服务端的保护；

501 Not Implemented 功能未实现，有点”敬请期待的意思”

502 Bad GateWay 一般是网关或代理返回的错误，服务器工作正常；

503 Service Unavailable 表示服务端繁忙，暂时无法响应；但是它是一个临时状态，响应报文里通常还会有一个Retry-After 字段指示客户端可以在多久以后再次尝试发送请求。
